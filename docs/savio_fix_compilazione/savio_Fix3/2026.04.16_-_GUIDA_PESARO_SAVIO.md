## 2. Come sostituire i file nel progetto

### Percorso dei file nel progetto Android

```
savio/
└── app/
    └── src/
        └── main/
            └── assets/
                ├── areas_v1.json          ← sostituisci questo
                ├── catalog_v1.json        ← NON toccare
                ├── equivalences_v1.json   ← NON toccare
                ├── offers_current.json    ← sostituisci questo
                └── stores_pilot_v1.json   ← sostituisci questo
```

### Step by step in Android Studio

1. Nel pannello **Project** (sinistra), naviga su:  
   `app → src → main → assets`

2. Clic destro su `areas_v1.json` → **Replace File** → seleziona il nuovo `areas_v1.json`

3. Ripeti per `stores_pilot_v1.json` e `offers_current.json`

4. In alternativa via filesystem:
   ```bash
   # Dalla cartella del progetto savio/
   cp /path/to/savio_pesaro/areas_v1.json        app/src/main/assets/areas_v1.json
   cp /path/to/savio_pesaro/stores_pilot_v1.json app/src/main/assets/stores_pilot_v1.json
   cp /path/to/savio_pesaro/offers_current.json  app/src/main/assets/offers_current.json
   ```

5. **Invalida la cache** di Android Studio per essere sicuri:  
   Menu → **File → Invalidate Caches / Restart**

6. Rebuild:  
   Menu → **Build → Rebuild Project**

### Verifica rapida post-sostituzione

Avvia l'app → vai su AreaSelection → scrivi "Pesaro" o "61121" nel campo di ricerca.  
Devono apparire le 3 nuove aree: **Centro / Baia Flaminia**, **Nord / Muraglia**, **Sud / Villa Fastiggi**.

### Come usare i loghi nel codice Kotlin/Compose

#### Passo 1 — Aggiungere i file drawable

Copia i PNG/SVG nella cartella:
```
app/src/main/res/drawable/
```

Per SVG usa **Android Studio → res → New → Vector Asset** e importa il file SVG.  
Per PNG usa semplicemente copia/incolla nella cartella (usa `drawable-xxhdpi` per PNG ad alta risoluzione).

#### Passo 2 — Mappare la chain al drawable in Kotlin

In `SavioComponents.kt` o in un nuovo file `StoreLogoUtils.kt`:

```kotlin
import androidx.annotation.DrawableRes
import com.paolomarchionetti.savio.R

@DrawableRes
fun chainToLogoRes(chain: String): Int? = when (chain.lowercase()) {
    "conad"      -> R.drawable.ic_store_conad
    "lidl"       -> R.drawable.ic_store_lidl
    "carrefour"  -> R.drawable.ic_store_carrefour
    "eurospin"   -> R.drawable.ic_store_eurospin
    "md"         -> R.drawable.ic_store_md
    "esselunga"  -> R.drawable.ic_store_esselunga
    "coop"       -> R.drawable.ic_store_coop
    else         -> null   // null = mostra sigla testuale come fallback
}
```

#### Passo 3 — Usare il logo nella StoreRankingCard (ResultsScreen)

```kotlin
@Composable
fun StoreRankingCard(estimate: BasketEstimate) {
    val logoRes = chainToLogoRes(estimate.store.chain)

    Row(verticalAlignment = Alignment.CenterVertically) {
        if (logoRes != null) {
            Image(
                painter = painterResource(id = logoRes),
                contentDescription = estimate.store.name,
                modifier = Modifier
                    .size(40.dp)
                    .clip(RoundedCornerShape(8.dp)),
                contentScale = ContentScale.Fit
            )
        } else {
            // Fallback: box con sigla testuale
            Box(
                modifier = Modifier
                    .size(40.dp)
                    .clip(RoundedCornerShape(8.dp))
                    .background(MaterialTheme.colorScheme.secondaryContainer),
                contentAlignment = Alignment.Center
            ) {
                Text(
                    text = estimate.store.name.take(2).uppercase(),
                    style = MaterialTheme.typography.labelMedium,
                    color = MaterialTheme.colorScheme.onSecondaryContainer
                )
            }
        }
        Spacer(Modifier.width(12.dp))
        // ... resto della card
    }
}
```

## 9. Come aggiornare le offerte ogni settimana

### Workflow manuale (MVP)

Ogni settimana, solitamente tra lunedì e giovedì:

1. **Scarica i volantini PDF** dai siti ufficiali:
   - Conad: https://www.conad.it/volantini
   - Lidl: https://www.lidl.it/it/le-nostre-offerte.htm
   - Carrefour: https://www.carrefour.it/it/volantino
   - Eurospin: https://www.eurospin.it/volantino/
   - MD: https://www.mdspa.it/volantino

2. **Identifica i prodotti in offerta** che corrispondono alle categorie in `catalog_v1.json`

3. **Aggiorna `offers_current.json`**:
   - Cambia `valid_from` e `valid_to` (lunedì → domenica della settimana)
   - Aggiorna `price_eur` se il prezzo è cambiato
   - Mantieni lo stesso `id` se il prodotto è lo stesso
   - Aggiungi un nuovo `id` (es. `off_923`) per prodotti nuovi

4. **Sostituisci il file** nella cartella `assets/` del progetto

5. **In futuro (CDN)**: il file verrà caricato su GitHub Releases o Cloudflare Pages e l'app lo scaricherà automaticamente all'avvio.

### Struttura date

```json
"valid_from": "2026-04-21",
"valid_to":   "2026-04-27"
```

Usa sempre formato `YYYY-MM-DD` (ISO 8601).

### Campi obbligatori vs opzionali

| Campo | Obbligatorio | Note |
|-------|-------------|------|
| `id` | ✅ | Univoco, non riusare mai |
| `store_id` | ✅ | Deve corrispondere a un id in `stores_pilot_v1.json` |
| `product_category_id` | ✅ | Deve corrispondere a un id in `catalog_v1.json` |
| `product_name` | ✅ | Come appare sul volantino |
| `price_eur` | ✅ | Prezzo in euro, virgola come punto decimale |
| `valid_from` | ✅ | |
| `valid_to` | ✅ | |
| `source_type` | ✅ | Sempre `"LEAFLET"` per dati da volantino |
| `confidence_level` | ✅ | `"HIGH"` per dati da volantino ufficiale |
| `brand` | ⚪ | `null` se prodotto sfuso o MDD senza brand |
| `price_per_unit` | ⚪ | `null` se non disponibile |
| `requires_fidelity` | ✅ | `false` se il prezzo non richiede tessera fedeltà |
